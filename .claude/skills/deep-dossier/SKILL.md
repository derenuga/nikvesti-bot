# Глибоке досьє з «лисячої нори»

Реалізація `docs/ENTITY_LAYER_PLAN.md` §2.2 руками, в сесії Claude Code.
Еталон — досьє на Віталія Воронова (647 згадок → 409 про нього; решта — село
Вороновка, ФК «Вороновка», однофамільці). Пайплайн повторює той прогін.

**Скрипти:** `scripts/dossier/` (export_foxhole, prep_batches, reduce_prep,
build_index, assemble). **Вигрузка з нори:** через GitHub Action
`.github/workflows/foxhole-export.yml`, бо з пісочниці прямого TCP до Railway
немає (egress-проксі пропускає CONNECT, але глушить Postgres-трафік).

Робоча тека — у скретчпаді сесії; далі `$W`. Гілка — поточна робоча гілка.

---

## Крок 0. Зібрати вхідні дані

Питай Олега (одним повідомленням, не по одному):
1. **Хто фігурант** — ім'я та прізвище + 1–2 рядки контексту (посада, місто,
   роль). Контекст критичний: він іде в промт map-агентам для відсіву тезок.
2. **Публічний URL нори** — `DATABASE_PUBLIC_URL` з Railway (Postgres →
   Variables), бо доступ через разовий Action. Попередь: ключ засвітиться в
   історії одного коміту → **ротувати після прогону**.

Згенеруй `TSQUERY` сам: прізвище префіксною лексемою + укр/рос варіанти написання
через `|`. Приклади: `Воронов:*` · `Кім:* | Ким:*` · `Сєнкевич:* | Сенкевич:*`.
Тільки прізвище (імена в тексті розріджені; однофамільців відсіє map-фаза).
Для теми/об'єкта — ключові слова обома мовами (`стадіон:* & центральн:*`).

---

## Крок 1. Вигрузка з нори (GitHub Action)

Запиши файл-тригер (гілка — поточна робоча) і запусти Action:

```
dossier_export/REQUEST:
  TSQUERY=<згенерований вираз>
  DB_URL=<публічний URL нори>
  OUT_PREFIX=dossier_export/<slug>
```

Закоміть і запуш REQUEST. Action `foxhole-export` вигрузить статті й закомітить
`<slug>.ndjson.gz` + `<slug>_index.tsv` у ту саму гілку. Стеж за прогоном
(`mcp__github__actions_list` / `actions_get`), потім `git pull`.

**Одразу після появи вигрузки — прибери ключ:** `git rm dossier_export/REQUEST`,
коміт, пуш. Додай `*.ndjson.gz` і `*_index.tsv` у `dossier_export/.gitignore`
(сира вигрузка в репо не потрібна). Нагадай Олегу ротувати URL наприкінці.

Перевір число: у логу Action рядок `matched(<TSQUERY>)=<N>`. Це скільки згадок
прізвища всього (для Воронова було 647). Якщо N величезне (>1500) або майже
нуль — уточни TSQUERY з Олегом перш ніж палити токени.

---

## Крок 2. Пачки для map

```bash
cp dossier_export/<slug>.ndjson.gz $W/topic.ndjson.gz
SRC=$W/topic.ndjson.gz OUTDIR=$W/batches BATCH=25 TEXT_CAP=7000 \
  python3 scripts/dossier/prep_batches.py
```

Виведе `articles=<N> batches=<K>` і гістограму по роках. Далі — `K` map-агентів.

---

## Крок 3. Map-фаза (K субагентів паралельно)

Запусти **всі K** субагентів (`general-purpose`) одним повідомленням. Кожному —
свій `batch_NN.json`. Промт (підстав `<ФІГУРАНТ>`, `<КОНТЕКСТ>`, `NN`):

> Ти — map-фаза глибокого досьє про **<ФІГУРАНТ>** (<КОНТЕКСТ>; у різні роки
> посади/фракції можуть змінюватись; існують ОДНОФАМІЛЬЦІ та тезки — їх треба
> відокремити).
>
> Прочитай `$W/batches/batch_NN.json` — JSON-масив статей {id, date, title, url, text}.
> Для КОЖНОЇ статті визнач:
> - `person`: `"subject"` (це наш фігурант), `"other"` (інший носій прізвища —
>   вкажи в `who` хто/що саме: тезка, однофамілець, топонім, назва клубу тощо),
>   `"unclear"` (прізвище є, але не зрозуміло хто).
> - `who`: роль/посада фігуранта НА МОМЕНТ статті, дослівно як у тексті.
> - `facts`: 1–3 речення КОНКРЕТНИХ фактів про фігуранта, українською. ТІЛЬКИ те,
>   що текстуально є в статті. Прохідна згадка — так і напиши одним реченням.
>   Без оцінок і домислів.
> - `significance`: 1–5 (5 = стаття ПРО нього / він центральний фігурант події;
>   3 = помітна роль; 1 = прохідна згадка в переліку).
> Якщо текст порожній або прізвища немає в наявному фрагменті — `person:"unclear"`,
> `facts:"згадка за межами наявного фрагмента"`, `significance:1`.
>
> Запиши результат у `$W/facts/batch_NN.json` як JSON-масив
> {id, date, title, url, person, who, facts, significance} (створи теку якщо треба).
> Поверни фінальним текстом один рядок: "batch_NN: N статей, S subject, O other, U unclear".

**Ліміт токенів:** якщо частина агентів впаде на `session limit`, вони часто
встигають записати файл до падіння. Перевір `ls $W/facts/`, до-запусти лише
відсутні `batch_NN` (тим самим промтом). Не перезапускай уже готові.

---

## Крок 4. Звести факти й порізати на періоди

```bash
FACTSDIR=$W/facts WORKDIR=$W MAXP=5 python3 scripts/dossier/reduce_prep.py
```

Виведе `person={...}` (скільки subject / other / unclear — це головна цифра
досьє, «X згадок → Y про нього») і періоди `p1..pN` з числом статей.
Створює `$W/subject_all.json`, `$W/all_classified.json`, `$W/periods/pN.json`,
`$W/periods_meta.json`, порожню `$W/narrative/`.

---

## Крок 5. Reduce-фаза (N субагентів, по періоду)

Запусти N субагентів (`general-purpose`) одним повідомленням. Кожному — свій
`periods/pN.json`. Промт (підстав `<ФІГУРАНТ>`, `N`, назву періоду):

> Ти пишеш розділ глибокого журналістського досьє про **<ФІГУРАНТ>**. Reduce-фаза:
> дано відібрані мікрофакти зі статей архіву МикВісті за період **<назва>**.
> Прочитай `$W/periods/pN.json` — масив {id, date, title, url, person, who, facts,
> significance} (усі person="subject").
>
> Напиши розділ УКРАЇНСЬКОЮ у Markdown:
> 1. Хронологічний **наратив** (з підзаголовками `###` за смисловими блоками):
>    ким був, що робив, ключові епізоди, конфлікти, ініціативи, цитати. Зв'язно, не перелік.
> 2. **ПРАВИЛО ЧЕСНОСТІ:** кожне твердження — з конкретної статті. Лінкуй
>    markdown-посиланням анкором 2–4 слова прямо в реченні; URL бери з поля `url`.
>    НЕ вигадуй фактів поза `facts`.
> 3. Опорні точки — епізоди significance 4–5. Якщо статей у періоді багато (>60) —
>    не згадуй кожну, обери показові, решту узагальни з 2–3 лінками-прикладами.
> 4. Обсяг пропорційний періоду (200–1100 слів). Почни із заголовка
>    `## <назва періоду>: <влучна назва>`.
> Запиши у `$W/narrative/pN.md` (тільки Markdown).
> Поверни один рядок: "pN готово, ~M слів".

---

## Крок 6. Синтез шапки (1 субагент)

Один субагент (`general-purpose`) читає `$W/subject_all.json` (ролі по роках +
факти significance≥4) і пише `$W/meta.json`:

> Ти — фінальний синтез досьє про **<ФІГУРАНТ>**. Прочитай `$W/subject_all.json`
> (масив {date, who, facts, significance}) і, за потреби, `$W/all_classified.json`
> (щоб описати однофамільців — там person="other" з полем who).
> Склади JSON і запиши в `$W/meta.json`:
> {
>   "subject": "<Ім'я Прізвище у називному>",
>   "identity": "одне речення — хто це, з траєкторією ролей у стислому вигляді",
>   "trajectory": [{"period":"РРРР–РРРР","role":"посада/статус","note":"1 рядок контексту"}, ...],
>   "open_questions": ["3–5 зачіпок для копання — недоведені сюжети, обірвані історії,
>       нез'ясовані переходи; кожна ГРУНТОВАНА на фактах, як зачіпка, не як звинувачення.
>       Можна з markdown-лінком на статтю."],
>   "homonym_note": "1–2 речення: хто ще носить це прізвище в архіві (тезки-люди з
>       іменами/посадами, топоніми, клуби) — узагальни person=other."
> }
> Тільки текстуально підтверджене. Поверни один рядок: "meta готово".

---

## Крок 7. Зібрати артефакти

```bash
WORKDIR=$W SUBJECT="<Ім'я Прізвище>" OUT=dossier_export/DOSSIER_<slug>.md \
  python3 scripts/dossier/build_index.py      # (можна .md; або одразу індекс у HTML нижче)
WORKDIR=$W META=$W/meta.json OUT=dossier_export/DOSSIER_<slug>.html \
  python3 scripts/dossier/assemble.py
WORKDIR=$W SUBJECT="<Ім'я Прізвище>" OUT=dossier_export/INDEX_<slug>.md \
  python3 scripts/dossier/build_index.py
```

`assemble.py` уже кодує не-ASCII у HTML-сутності (Safari без charset-заголовка
інакше дає mojibake) — окремо нічого робити не треба.

Перевір: `grep -c '[^ -~]' DOSSIER_<slug>.html` має бути 0; число лінків
розумне; шапка/траєкторія/питання не порожні.

---

## Крок 8. Віддати

1. `Artifact` — опублікуй `DOSSIER_<slug>.html` (favicon 🦊, опис українською).
2. `SendUserFile` — надішли HTML (`display:"attach"`) і `INDEX_<slug>.md`.
3. Закоміть у гілку `DOSSIER_<slug>.html` + `INDEX_<slug>.md` (сиру вигрузку —
   ні, вона в `.gitignore`). Пуш.
4. Підсумуй Олегу: головна цифра (**X згадок → Y про нього**), що дав наратив,
   орієнтовна вартість токенів, і **нагадай ротувати URL нори**.

## Орієнтири й запобіжники

- **Вартість:** ~25 статей/пачка. Для ~650 статей — ~26 map + ~5 reduce + 1 синтез
  ≈ 1,8 млн токенів субагентів (чистий прогін). Через API (Haiku map + Sonnet
  reduce) це ~$1 — оцінка `/dossier_deep` з плану.
- **Однофамільці — суть цінності.** Головний доказ методу: `X згадок → Y про
  людину`. Обов'язково винеси це число і розбір `other` у шапку.
- **Правило чесності беків** діє наскрізь: факт без лінка не виходить у наратив.
- **Не палити даремно:** перевір `matched=N` (крок 1) перед map. N>1500 — уточни
  запит; N<10 — можливо, не той TSQUERY або людини майже немає в архіві.
- **Безпека:** URL нори — живий пароль. Разовий Action, чистка REQUEST одразу,
  ротація наприкінці. На читання, нічого не пишемо в нору.
