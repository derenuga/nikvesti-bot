# Таблиця аналітики соцмереж і сайту (social_sheet.py)

Бот сам веде Google Spreadsheet **«Аналітика МикВісті»** — щомісячний знімок
метрик сайту й соцмереж. Заміна ручної «архаїчної» таблиці, яку редакція вела
з блоками по мережах і місяцями в колонках.

**Таблиця:** `SOCIAL_SPREADSHEET_ID` = `1KNkxqN8ru4c2ez-x3nw9sEW-lXm562VdfJ3EEdbjGZk`
(створена Олегом 14.07.2026, розшарена на service account з GA4_CREDENTIALS як
Editor — SA не може створювати файли, лише листи всередині; та сама схема, що
тендерна таблиця в sheets.py).

## Структура

- **Лист на рік** («2025», «2026»…), створюється ботом автоматично при першому
  записі місяця цього року. Місяці — РЯДКАМИ, метрики — колонками (best
  practice усіх шаблонів соцзвітів: набір метрик фіксований, таблиця росте вниз,
  бот просто заповнює рядок).
- **Блоки всередині листа** (рядки фіксовані, формули зав'язані на адреси):
  - 3–17 🌐 Сайт: користувачі/Δ/сеанси/перегляди/Δ (GA4) + кліки Search/News/
    Discover + частка Discover (Search Console). AI Overviews Google в API
    окремим типом НЕ віддає — сидять у web/Search.
  - 19–33 📘 Facebook: підписники/±, перегляди (page_media_view)/Δ, взаємодії/Δ,
    пости, топ-допис лінком.
  - 35–49 📷 Instagram: підписники/±, перегляди/Δ, охоплення, взаємодії/Δ, пости.
  - 51–65 ✈️ Telegram: підписники/±, сер. охоплення поста/Δ, пости, перегляди
    за місяць, ERR (= сер. охоплення / підписники).
  - 67–81 ▶️ YouTube: підписники/±, перегляди відео/Δ, час перегляду (год),
    контент, CTR. Бот заповнює підписники/перегляди/контент через YouTube
    Data API (див. таблицю джерел); час перегляду і CTR — вручну до OAuth.
  - 83–97 🎵 TikTok: підписники/±, перегляди відео/Δ, охоплення,
    вподобайки, поширення, коментарі. Бот заповнює підписники + вовлеченість
    (перегляди/лайки/поширення/коментарі по відео місяця) через TikTok
    Display API (див. таблицю джерел); охоплення (F) — вручну (API не віддає).
  - 99–113 💜 Viber: підписники/±, активні користувачі/Δ, надіслано
    повідомлень. Viber — БЛОК-КАРКАС (колонки — за метриками старої ручної
    таблиці): повне оформлення, дельти-формули, підсумки, спарклайни — але
    бот його не заповнює, поки не підключений API; числа приїхали міграцією,
    нові — вручну, формули оживають самі. Старі листи (створені зі stub-
    рядками на 67–69) добудовуються на місці автоматично
    (`_upgrade_year_sheet`, маркер A68 == «Місяць»).
- **Рядок «Підсумок {рік}»** у кожному блоці — ЖИВІ ФОРМУЛИ (SUM/AVERAGE +
  для підписників «останнє відоме значення» через `LOOKUP(9^99;діапазон)` —
  саме так, НЕ Excel-трюк `LOOKUP(2;1/(rng<>""))`, який у Google Sheets без
  ARRAYFORMULA тихо дає помилку; + YoY через INDIRECT на лист попереднього
  року; IFERROR лишає порожньо, поки того листа немає). Нічого «вставляти в
  кінці року» не треба — рахується саме з першого місяця, працює і на
  частково заповнених роках (2022–2023 з точковими знімками).
- **Дельти MoM — формули** (січень порівнюється з груднем минулорічного листа).
  Бот пише лише сирі числа; стрілки ▲▼ і колір малює custom number format
  (`[Color 10]▲ …;[Color 9]▼ …`) — у клітинці чисте число.
- **Спарклайн року** — формула SPARKLINE у злитій клітинці колонки «Тренд»
  (ключова метрика блоку: сайт/FB/IG — перегляди, TG — підписники).
- **Два вбудовані графіки** праворуч (M): «Підписники по мережах» (лінії,
  брендові кольори FB #1877F2 / IG #C13584 / TG #229ED9) і «Перегляди сайту по
  місяцях» (стовпчики). Створюються раз при створенні листа, діапазони на всі
  12 місяців — оновлюються самі.

**Лого мереж:** у клітинці A шапки кожного блоку — офіційний PNG з Wikimedia
Commons (стабільні thumb-URL у `LOGOS`, хотлінк дозволений) на білому чипі,
формулою `=IFERROR(IMAGE(url);емодзі)` — якщо Commons перейменує файл,
замість діри буде емодзі. Назва блоку — у злитій B:K. Листи, створені до
лого-чипів, ретрофітяться самі (`_upgrade_band_logos`).

**Теми:** дві палітри в `THEMES` — `dark` (дефолт, як нічна палітра
веб-макета: холст #17181B, світлий текст, підсвічені бренд-кольори
спарклайнів/графіків, стрілки ▲▼ яскравими [Color4]/[Color3]) і `light`.
Тема задається `/sheet_format [рік] [dark|light]` або env `SOCIAL_SHEET_THEME`.
Разом з оформленням виставляється **SpreadsheetTheme** (TEXT/BACKGROUND/LINK/
акценти) — інакше графіки лишились би білими з чорними осями, а лінки
темно-синіми; тема документа глобальна для всіх листів. Графіки при перекаті
перестворюються (старі видаляються) — щоб зміна теми перефарбовувала і їх.

**Локаль:** формули писані під `uk_UA` (`;` між аргументами, `\` в масивах
SPARKLINE) — `_ensure_locale()` примусово ставить locale=uk_UA/Europe/Kiev при
кожному записі.

## Джерела даних за місяць

| Блок | Звідки | Глибина бекфілу |
|---|---|---|
| Сайт GA4 | `get_stats(client, start, end)` напряму (не daily_stats — GA4 тримає всю історію, один запит) | уся історія ресурсу |
| Сайт SC | `searchanalytics.query` по типах web/discover/googleNews, тоталі за місяць | ~16 місяців |
| Facebook | insights `period=day` + since/until (місяць < вікна 93 дні), суми; пости+топ — пагінація `/posts` (рахуються ВСІ пости сторінки, на відміну від тижневих звітів, де лише з лінком на сайт) | ~2 роки (що Meta не віддасть — пропуститься; page_media_view існує з ~2025) |
| Instagram | insights `metric_type=total_value` + since/until (тотал за діапазон; якщо місяць не влазить — сума двох половин), пости — пагінація `/media` | скільки віддасть Meta |
| Telegram | стрічка t.me/s: старт з оцінки message_id кінця місяця по якорях `CALIBRATION_ANCHORS` (+250 запасу), гортання вниз до виходу за початок місяця; підписники — веб-прев'ю `t.me/{channel}` («41 012 subscribers»), фолбек — округлений лічильник /s | уся історія каналу, АЛЕ перегляди — поточні накопичені (для минулого місяця ≈ фінальні; t.me округлює 12.3K) |
| YouTube | **YouTube Analytics API через OAuth** (`handlers/youtube_analytics.py`): підписники — поточний лічильник Data API (тим самим токеном); перегляди відео (D) і години перегляду (F) за місяць — точний агрегат Analytics. Контент (G) і CTR (H) — вручну (API їх стабільно не віддає). Налаштування — OAuth client + refresh token власника каналу (docstring модуля) | **повна історія**: `/youtube_backfill` тягне помісячний розріз (dimension=month) за всю історію каналу одним запитом — перегляди й години назад до старту; підписники по місяцях — реконструкція від реального якоря в таблиці через gained/lost |
| TikTok | **TikTok Display API через OAuth** (`handlers/tiktok_analytics.py`): підписники (B) — поточний лічильник; перегляди/лайки/поширення/коментарі (D/G/H/I) — сума по ВІДЕО, опублікованих у місяці (їх lifetime-лічильники). Охоплення (F) — вручну (публічного API немає). TikTok РОТУЄ refresh token → бот тримає актуальний у `storage.tiktok_oauth`, env лише сідує | **історії немає** — API віддає лише знімок «на зараз», помісячного розрізу минулого нема; тільки вперед (минуле — з міграції) |

**Підписники заднім числом недоступні ніде** — бекфіл колонки «Підписники» не
чіпає, вони накопичуються живими знімками вперед (+ перенесення зі старої
таблиці, див. нижче).

**Дзеркала Telegram:** `telegram_stats.TG_WEB_BASES = ["https://t.me",
"https://telegram.me"]` — t.me періодично блокують (07.2026 Чорногорія
заблокувала домен), робочий домен запам'ятовується на процес. Перевірено на
живому HTML: у пісочниці розробки t.me не відповідає, telegram.me працює,
парсинг блоків/лічильників — ок.

## Запуск

- **Авто:** 1-го числа о 10:30 (`run_monthly_snapshot`) — знімок попереднього
  місяця, короткий звіт Олегу в приват, збій → notify_error.
- **/sheet_snapshot [YYYY-MM]** — вручну (дефолт попередній місяць).
  Ідемпотентно: клітинки рядка просто перезаписуються; блок, чиє джерело
  впало, пропускається (існуючі числа не затираються).
- **/sheet_backfill [місяців]** — історія (дефолт 36, кап 120), від давніх до
  свіжих, прогрес у повідомленні. «Часткові/порожні» місяці — норма для давніх
  періодів (ліміти джерел вище).
- **/sheet_format [рік]** — примусово перекатити статику й оформлення листа
  (дані місяців не чіпає). Бот і сам лікує «голий» лист при наступному записі:
  якщо на листі майже немає злиттів — отже batchUpdate оформлення колись упав
  (він атомарний: один битий запит = нуль форматів). Так і сталось у першій
  версії: заголовок зливався A1:K1 через закріплену колонку A — Sheets
  забороняє «merge frozen and non-frozen columns», тому 🦊 живе в A1, а
  назва/примітка зливаються від B. Оформлення й графіки — окремі batchUpdate,
  щоб збій одного не валив інше і в помилці було видно, який крок упав.

## Перенесення старої таблиці («МикВісті SMM») — зроблено

`social_sheet_legacy.py` + `/sheet_migrate_legacy`: разовий перенос історії зі
старої ручної таблиці SMM (Google Sheets `1MDov5UBfTz1L…`) — 29 місяців
(2024-02 … 2026-06), розпарсених Claude-сесією 14.07.2026 у
`data/legacy_smm.json` (668 значень). Переносяться: підписники всіх мереж
(їх API заднім числом не віддають), перегляди/охоплення/взаємодії IG і FB,
сер. охоплення TG, і ЦІЛКОМ YouTube/TikTok/Viber (під них блоки-каркаси).
Парсер відсіяв несвіжі копії (липень–грудень на листі «Статистика 2026» були
дублем 2025-го), нормалізував «148к»/«7,3 тис.»/нерозривні пробіли, звів CTR
YouTube до частки; спот-чек звірено з живим скріном (березень 2026).
Стик методологій: до 2026-06 — числа старої таблиці, з 2026-07 — числа бота
з API (Δ на стику липня один раз буде «кривою»). Січень 2024 у старій
таблиці відсутній (звітність починалась з лютого). Команда ідемпотентна.

## Обмеження / відомі особливості

- «Індекс цитування» і охоплення TG як у TGStat — лише через їх платний API,
  не реалізовано.
- IG reach при сумуванні половин місяця трохи завищується (унікальність).
- FB page_media_view: якщо Meta відхилить period=day — перегляди лишаться
  порожніми (у логах буде причина), взаємодії page_post_engagements мають
  довгу історію.
- Перший реальний тест GA4/SC/FB/IG-гілок — /sheet_snapshot на проді
  (у пісочниці немає кредів), Telegram-гілка перевірена на живому HTML.
