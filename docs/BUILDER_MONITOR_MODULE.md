# Монітор білдера головної — `handlers/builder_monitor.py`

Нагадує редакції оновити головну сторінку сайту, коли вона застоялась, а свіжі
власні матеріали вже вийшли. Перший модуль, побудований поверх прямого доступу
до production-БД сайту (`handlers/db.py`, крок A `LONG_TERM_VISION.md §2.4`).

## Що таке «білдер»

Білдер — «конструктор» першого екрана nikvesti.com: журналісти вручну верстають
туди матеріали, застосовуючи шаблони. Зберігається в таблиці `options`.

## Схема БД (розвідано 03.07.2026 через /dbquery + phpMyAdmin)

Знання нижче — щоб наступні DB-модулі не передискаверювали структуру.

### `options` — стан білдера (і всі опції сайту)

| Колонка | Тип | Примітка |
|---|---|---|
| `name` | varchar, PK | у білдера завжди `'builder'` |
| `namespace` | varchar, PK | блок першого екрана: `a1,a2,a3,b1,c3,t1,t2…` |
| `value` | longtext | серіалізована JSON-верстка блоку (великий, НЕ чіпати) |
| `type` | varchar | — |
| `date` | int | **unix-час останнього збереження** |

Усі блоки білдера зберігаються одним збереженням → `date` у всіх рядків
`name='builder'` однаковий. **`builder_last = MAX(date) WHERE name='builder'`.**

### `nodes` — матеріали (72 колонки, ключові)

| Колонка | Тип | Примітка |
|---|---|---|
| `id` | int PK | autoincrement |
| `type` | varchar | `news`, тощо |
| `status` | int | `1` = опубліковано, `0` = чернетка |
| `title_ua` / `title` | varchar | заголовок (беремо `title_ua`, фолбек `title`) |
| `own_material` | tinyint | `1` = наш власний матеріал (не агентський/рерайт) |
| `owner_id` | int | → `users.id` (автор); колонка `author` порожня, НЕ використовувати |
| `published` | int | **unix-час публікації; БУВАЄ В МАЙБУТНЬОМУ** (відкладені пости) |
| `created` / `updated` | int | створення / останнє редагування |
| `slug_ua` / `category` | varchar | для побудови URL (наразі не використовуємо) |

⚠️ `published` гейтимо по `<= now` — інакше рахували б ще неопубліковані
відкладені матеріали (`status=1`, але `published` попереду).

### `users` — автори/редактори

`id`, `first_name`, `last_name`, `middle_name` (+ `_ru`/`_en`), `username` (логін,
НЕ ім'я). Відображуване ім'я = `first_name` (+ `last_name` де треба).

### `logs` — аудит адмін-дій (для «хто оновив білдер»)

`id`, `user_id` (varchar), `related_record`, `action`, `date` (unix), `url`,
`incoming_data`. Збереження білдера логується як **`url = '/admin/builder/save/'`**,
причому `logs.date` збігається з `options.date`. Останній такий запис → редактор.
`url` не проіндексований, тож запит важкий — робимо його лише при формуванні
повідомлення (алерт/`/builder`), не в кожній перевірці.

## Логіка алерту

```
builder_last = MAX(options.date WHERE name='builder')
if now - builder_last > BUILDER_STALE_HOURS (2 год):
    news = nodes WHERE own_material=1 AND status=1
                 AND published > builder_last AND published <= now
    if len(news) >= MIN_FRESH_NEWS (2):
        якщо кулдаун ALERT_COOLDOWN_HOURS (2 год) минув → пост у чат редакції
```

**Самозгасання:** щойно редактор оновить білдер, `builder_last` стрибає на now →
`news` порожніє → умова зникає. Кулдаун (`storage.builder_monitor.last_alert_at`)
страхує від повторів, поки редакція не відреагувала.

**Куди постить:** чат редакції `CHAT_ID` (не в приват). Автор — до кожного
заголовка, тому без узгодження дієслова за родом/числом.

## Пороги (константи зверху модуля)

| Константа | Значення | Що це |
|---|---|---|
| `BUILDER_STALE_HOURS` | 2 | скільки білдер має «висіти» без оновлення |
| `MIN_FRESH_NEWS` | 2 | скільки власних новин має вийти за цей час |
| `ALERT_COOLDOWN_HOURS` | 2 | не частіше одного нагадування |
| `MAX_TITLES` | 6 | скільки заголовків у пості (решта — «…і ще N») |
| `BUILDER_SAVE_URL` | `/admin/builder/save/` | URL збереження білдера в `logs` |

## Розклад

`:10` і `:40` у робочі години **9–21 (Києвом)** — scheduler у `Europe/Kiev`, тож
cron-година = київська. Не збігається з рештою задач на `:00/:05/:15/:30/:35`.

## Команди

| Команда | Дія |
|---|---|
| `/builder` | Діагностика: коли/хто оновлював, скільки власних новин відтоді, чи спрацював би алерт. Тільки редакція. |
| `/builder_test` | Постить зразок нагадування в чат редакції зараз, в обхід порогів (останні 3 власні новини). Тільки редакція. |

## Стійкість

Без `DB_*` env — тихо пропускає (модуль опційний). Виняток → `notify_error`
(алерт Олегу в приват), як у решти моніторів.
