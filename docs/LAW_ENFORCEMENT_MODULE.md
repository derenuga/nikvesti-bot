## МОДУЛЬ ПРАВООХОРОНЦІ (моніторинг новин правоохоронних органів) — детальний опис

**Належить до проекту:** Лис Микита — бот редакції МикВісті (див. PROJECT_CONTEXT.md).

### Що робить
Три рази на день перевіряє сторінки новин правоохоронних органів Миколаївщини, знаходить нові пресрелізи і надсилає їх одним повідомленням в канал "🦊 Микита винюхав".

### Підключені джерела
- ✅ **Миколаївська обласна прокуратура** — myk.gp.gov.ua/ua/news.html

### Файли коду
```
handlers/law_enforcement.py — парсери, логіка, форматування, відправка
handlers/storage.py — ключ "document_ids": {"prokuratura": [...]}
```

### Команди бота
- /law — перевірити всі джерела вручну

### Env-змінні
- DOCUMENTS_CHAT_ID — канал для постів (fallback на PROZORRO_CHAT_ID)
- Зараз: -1004322862192

### Розклад
Три рази на день: 10:00, 13:00, 16:00 (Europe/Kiev).

---

### Технічні деталі по джерелах

**1. Миколаївська обласна прокуратура (myk.gp.gov.ua)**
- URL: `https://myk.gp.gov.ua/ua/news.html`
- Структура: `section.default > ul > li > a.blue_bold[href="/ua/news.html?...&id=424328"] > p > span.grey_bold (дата) + текст (заголовок)`
- ID: числовий з query-параметра `id=` у href (`id=424328` → `"424328"`)
- Дата: `span.grey_bold` всередині `<p>`, формат `26.06.2026`
- Заголовок: текст `<p>` після видалення `<span>` і `<br>`
- На сторінці ~6 новин, пагінація через `?fp=0`, `?fp=10` тощо (перевіряється тільки перша сторінка)
- User-Agent: `NikVesti-Bot/1.0`

**Формат поста:**
```
⚖️ Прокуратура Миколаївщини

Заголовок новини (посилання) (26.06.2026)
Заголовок новини 2 (посилання) (26.06.2026)
```

**Baseline при першому запуску:**
N=1 — зберігаємо всі новини крім 1 найновішої як бачені, 1 найновішу відправляємо одразу.
N=1 обрано бо пресрелізи прокуратури виходять рідше ніж документи міськради — одного достатньо для перевірки формату.

**Storage:** `/data/prozorro_state.json`, ключ `"document_ids"`:
```json
{
  "document_ids": {
    "prokuratura": ["424328", "424321", ...]
  }
}
```
Використовує ті самі функції `get_seen_document_ids` / `save_seen_document_ids` що й модуль Документи.

**Архітектура конфіг-driven:** нове джерело = один рядок у `LAW_ENFORCEMENT_SOURCES` у `law_enforcement.py`. Якщо структура HTML нового сайту відрізняється — пишеться окремий парсер і вказується в полі `"parser"`.

### Відомі обмеження

- **Пагінація** — перевіряється тільки перша сторінка (~6 новин). Якщо між 10:00 і 13:00 вийде більше 6 новин — надлишок пропуститься (на практиці малоймовірно).
- **Без фільтрації** — всі новини відправляються, включаючи адміністративні (візити, наради тощо). Фільтрація не потрібна — редакція вирішує що цікаво.
