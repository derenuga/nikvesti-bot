## МОДУЛЬ КОНКУРЕНТИ (моніторинг новин інших миколаївських медіа) — детальний опис

**Належить до проекту:** Лис Микита — бот редакції МикВісті (див. PROJECT_CONTEXT.md).

### Що робить
Щогодини парсить головні сторінки сайтів-конкурентів, фільтрує миколаївські новини по словнику ключових слів і надсилає нові в чат редакції з AI-підводкою у стилі Лиса Микити.

### Підключені джерела
- ✅ **news.pn** — головна сторінка `https://news.pn/uk/` (фільтр по часу 3 год — захист від підняття старих новин), `filter_local=True`
- ✅ **novosti-n.org** — лівий фід головної `https://novosti-n.org/ua/` (без фільтру по часу — все нове по ID), `filter_local=True`
- ✅ **Суспільне Миколаїв** — загальний RSS `https://suspilne.media/rss/all.rss`, фільтр по `/mykolaiv/` в URL, `filter_local=False` (всі новини вже миколаївські, словник не застосовується)

> **Історія:** Суспільне додано 21.06.2026, але випадково вирізане з коду 22.06 при повній перезаливці `competitors.py` (коміт `565ca98` — блок Суспільного втрачено, останні новини Суспільного в чаті — 22.06). Відновлено разом з механізмом `filter_local` (гілка `suspilne-news-monitoring-gap`).

### `filter_local` — прапорець джерела
- `True` (news.pn, novosti-n) — сирий фід усього міста/країни, фільтруємо по `LOCAL_KEYWORDS` (словник миколаївських тем) і на baseline, і на нових новинах.
- `False` (Суспільне) — фід уже звужений до Миколаєва (RSS відфільтрований по `/mykolaiv/` в парсері), тож словник **не застосовується** — інакше губилися б миколаївські новини без ключового слова в заголовку (напр. «Синоптики прогнозують нестійку погоду»).

### Файли коду
```
handlers/competitors.py — парсери, фільтрація, форматування, відправка, нічний буфер
handlers/storage.py — ключі "competitor_ids": {"news_pn": [...], "novosti_n": [...], "suspilne_mk": [...]} і "competitor_night_buffer": [...]
```

### Технічні деталі

**Парсинг news.pn:**
- Структура: `div.hentry > span.t (час) + a[href=/uk/категорія/ID] > span (заголовок)`
- ID новини: числовий в кінці URL (`/uk/criminal/345266` → `"345266"`)
- User-Agent: `NikVesti-Bot/1.0` (перевірено — 200 OK з Railway)
- Доступ з Railway: підтверджено, HTTP 200
- Час на сторінці приходить в UTC (Railway-запит) — код конвертує в київський (+3). При переїзді на UA-сервер — прибрати (коментар у `parse_news_pn`)

**Парсинг novosti-n.org:**
- Контейнер: `div.mewsCity > div.newsList__item.ddd.hentry`
- Час: `div.newsList__time > span` (буває "Вчора о 23:18" або "08:00")
- ID: числовий в кінці URL (`/ua/news/Nazva-340345` → `"340345"`)

**Фільтрація (словник ключових слів):**
Навчений на реальних заголовках з news.pn, novosti-n.org і nikvesti.com sitemap.
Поточна версія:
```python
LOCAL_KEYWORDS = re.compile(
    r'Миколає|Миколаїв|миколаїв|миколаївськ|'
    r'Інгул|Намив|Парутин|Слобідськ|'
    r'Галицинів|Снігурівк|Вознесенськ|Баштанськ|Первомайськ|'
    r'Очак|Южноукраїнськ|'
    r'Кім|Сєнкевич|'
    r'Куцуруб|Новоодеськ|Мертвовод|'
    r'Корабельн',
    re.IGNORECASE
)
```
Словник продовжує навчатись на реальних заголовках — доповнювати при знаходженні нових пропусків.

**Baseline:** при першому запуску джерела зберігаємо всі поточні ID як бачені, а 3 останні локальні новини одразу відправляємо — щоб перевірити що парсинг і відправка працюють після деплою.

**Storage:** `/data/prozorro_state.json`, ключ `"competitor_ids"`:
```json
{"competitor_ids": {"news_pn": ["345266", ...], "novosti_n": ["340345", ...]}}
```

**Важлива деталь:** зберігаємо seen_ids для ВСІХ нових новин (не тільки миколаївських) — щоб національні новини не з'являлись повторно при наступних перевірках.

**Формат повідомлення:**
```
🔍 Новини інших миколаївських медіа на регіональну тематику

📰 NEWS.PN
09:11 — Заголовок новини (посилання)
```
Без AI-підводки — системний вигляд, за рішенням Олега (07.2026 підводку спробували і прибрали).

**Нічний режим (00:00–07:00 за Києвом):** нові новини не шлються одразу, а збираються в буфер у storage (ключ `competitor_night_buffer`) і приходять одним дайджестом «🌙 Новини конкурентів за ніч» з першою ранковою перевіркою (07:15). Буфер переживає рестарт бота. Seen ids зберігаються одразу при знаходженні — дублікатів у дайджесті не буває.

### Дослідження RSS/sitemap конкурентів

**news.pn:**
- RSS відсутній (сайт на Vue.js/Nuxt, жодного `<link rel="alternate">` в HTML)
- Sitemap і robots.txt: HTTP 403 з будь-якого автоматизованого клієнта
- Google News RSS (`site:news.pn Миколаїв`): протестовано, але нереліабельний — пропускає частину локальних новин (підтверджено на прикладі новини про УСР)
- **Рішення:** прямий парсинг головної сторінки

**novosti-n.org:**
- Структура: URL новин `/ua/news/Назва-ЧИСЛОВИЙID`
- Прямий доступ підтверджений (HTTP 200)
- RSS: не перевірявся детально, але парсинг HTML є надійнішим варіантом
