# Бюджет Миколаєва в норі: версії плану і рішення про зміни

Модуль: `handlers/budget_revisions.py`. Схема `budget` у Postgres бота (нора).
Це «задание №2» бюджетного напряму. **Задание №1 (щомісячні снапшоти виконання
+ завантажувач xlsx звітів) у репо ще не реалізоване** — цей модуль від нього
не залежить і існуючих таблиць не чіпає, але звірка «план ревізії vs
annual_plan із найближчого снапшота» додасться лише разом із №1.

## Модель

Горсовет вносить зміни рішеннями сесії «викласти додаток у новій редакції» —
повною заміною без дельт. До кожного рішення департамент фінансів готує пакет
**«Порівняльні таблиці»** (xlsx): на одному листі лівий блок «Чинна редакція»
і правий «Запропонована редакція з урахуванням змін», порядково вирівняні.
Дельти = права − ліва. Ці xlsx — єдине джерело завантаження (PDF не парсимо).
На сайт міськради вони не викладаються — приходять у пакеті матеріалів сесії
(редакція отримує їх у Drive, теки типу «З голосу_29.01.2026»).

Зміни плану бувають і без сесії (розпорядження мера по трансфертах, виконком
по постанові КМУ №18) — тому сума дельт рішень не зобов'язана пояснювати всю
динаміку плану; снапшоти (№1) — первинний факт, рішення — атрибуція.

## Таблиці (схема budget)

| Таблиця | Що |
|---|---|
| `plan_revision` | Реєстр редакцій: рік, номер рішення, дата, kind original/amendment, effective_order (0 = original), UNIQUE(рік, order) |
| `plan_expenditure_line` | Видатки (Додаток 3): КПКВК/ТПКВК/КФК + 12 грошових колонок, is_unit_total для підсумків розпорядників (XX00000/XX10000), UNIQUE(revision, kpkvk) |
| `plan_revenue_line` | Доходи (Додаток 1): код (8 зн.), total/general_fund/special_fund/special_dev_budget, UNIQUE(revision, code) |
| `plan_headline` | Показники текстової частини рішення (доходи/видатки/резервний фонд/борг/чисельність) — вручну через /budget_headline |
| `revision_validation_issue` | Розбіжності «чинної редакції» документа зі збереженою попередньою ревізією (реальність: одруківки депфіну; завантаження НЕ зупиняють) |

В'юхи: `v_plan_amendments` (дельти видатків між сусідніми ревізіями, без
підсумків розпорядників, з прапором нових програм), `v_plan_revenue_amendments`
(те саме для доходів), `v_plan_disappeared` (програми, що зникли з нової редакції).

## Парсер (перевірено на реальному файлі s-fi-005 Додаток 3)

- Межа блоків — **повторний старт нумерації колонок з «1»** (рядок «1 2 … 16 |
  1 2 … 16»), не фіксований номер колонки. Один блок = додаток вихідного
  рішення (kind='original').
- Ширина блоку: видатки 16, доходи 6 — інша ширина = помилка «структура
  змінилась» (краще впасти голосно, ніж завантажити криво).
- Коди в xlsx — числа без ведучих нулів (210160.0), zero-pad: КПКВК 7, ТПКВК 4,
  КФК 4, код доходу 8; код бюджету 1.4549E9 → '1454900000'.
- Рядок «УСЬОГО» (коди 'Х') не зберігається — контроль: сума завантажених
  програмних рядків (без unit totals) має збігтися з ним копійка в копійку.
- Нова програма = порожній лівий блок рядка; у валідацію не пише, просто
  вантажиться з прапором is_new_program у в'юсі.

## Логіка завантаження

1. Перша порівняльна таблиця року, ревізій немає → ліва частина зберігається
   як ревізія kind='original' (notes='reconstructed from comparison table',
   номер — з `base=`, дефолт 'original'), права — amendment #1.
2. Другий файл того ж рішення (інший додаток) → та сама ревізія, доливається.
   Якщо original був реконструйований без цього додатка — доливається і він.
3. Повторне завантаження того ж файлу → ревізія НЕ дублюється, рядки таблиці
   і validation issues перезаписуються (ідемпотентно).
4. Ліва частина валідується проти ревізії effective_order−1: кожна розбіжність
   (значення поля або відсутність рядка) → `revision_validation_issue`,
   завантаження продовжується.

## Команди

| Команда | Що |
|---|---|
| `/budget_load <рік> <номер> <дата> [base=50/26]` | Підпис до xlsx-документа (або reply на нього). Приклад: надіслати боту «s-fi-001 Додаток 3 Порівняльна таблиця.xlsx» з підписом `/budget_load 2026 50/123 29.01.2026 base=50/26` |
| `/budget_status` | Ревізії по роках: рядки, суми, розбіжності валідації |
| `/budget_headline <рік> <номер> ключ=значення …` | Показники текстової частини рішення вручну (revenue_total, expenditure_total, reserve_fund, debt_limit, guaranteed_debt_limit, staff_count …) |

Доступ: `ALLOWED_USER_IDS` (як analytics_backfill). Без `BOT_DATABASE_URL`
модуль каже, що нора не налаштована, і нічого не робить.
