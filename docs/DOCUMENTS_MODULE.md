- ✅ **Офіційні оголошення міськради Миколаєва** — mkrada.gov.ua/news/?c=2
- ✅ **Розпорядження голови ОВА** — mk.gov.ua/ua/oda/order/ (фільтр за останні 3 дні)
- ✅ **Проєкти рішень Миколаївської обласної ради** — mk-oblrada.gov.ua/proekty-rishen?group_id=14
- ✅ **Проєкти рішень Миколаївської міської ради** — mkrada.gov.ua/content/proekti-rishen-miskradi.html


### Технічні деталі по джерелах

**1. Розпорядження міського голови (mkrada.gov.ua)**
- Структура: `div.news_line_item > a[href=/documents/ID.html] + div.date > strong(номер)`
- ID: числовий з URL (`/documents/51982.html` → `"51982"`)
- Файл документа: сам HTML (не PDF/DOC) — превью в Telegram не генерується
- Формат поста: `№ 275р від 17.06.2026 / Назва документа`

**2. Розпорядження голови ОВА (mk.gov.ua)**
- Структура: `a.list-group-item[href=?doc_id=18165] > h4 (дата+номер) + p.list-group-item-text (назва)`
- ID: числовий з query-параметра `?doc_id=18165`
- Запит: фільтр за останні 3 дні (`s_date`, `e_date`) — страховка при перезапуску бота
- Відомий баг: в кінці назви іноді є "Зареєстровано в Одеському міжрегіональному управлінні..." — треба обрізати (беклог п.6)
- Формат поста: `№ 244-р від 15.06.2026 / Назва документа`

**2а. Офіційні оголошення міськради (mkrada.gov.ua/news/?c=2)**
- Структура: `div.news_line_item > a[href=/news/ID.html] + div.date` (без strong)
- Дата: злиплий формат "16 Червня 202616:15" — парситься регексом
- ID: числовий з URL `/news/25518.html`

**3. Проєкти рішень облради (mk-oblrada.gov.ua)**
- URL: `https://mk-oblrada.gov.ua/proekty-rishen?group_id=14` (group_id=14 = постійна сторінка "наступна сесія")
- Структура: `table.table-striped > tr > td[0] (назва + посилання на PDF) + td[1] (пояснювальна)`
- ID: md5 від повної назви документа (PDF не завжди є, тому URL файлу не є надійним ID)
- Пагінація: підтримується (`?page=N`), гортає всі сторінки
- Якщо PDF відсутній — позначається `(файл відсутній)` в пості
- Формат поста: `📃 Про висунення... повна назва / Перелік: https://mk-oblrada.gov.ua/proekty-rishen`

**Baseline при першому запуску:**
Всі документи крім 10 найновіших зберігаються як "вже бачені". 10 найновіших одразу відправляються в канал — щоб перевірити що відправка і формат працюють правильно після деплою.

**Storage:** `/data/prozorro_state.json`, ключ `"document_ids"`:
```json
{
  "document_ids": {
    "mayor_orders": ["51982", "51981", ...],
    "ova_orders": ["18165", "18164", ...],
    "oblrada_decisions": ["1580639109c96e00", ...]
  }
}
```
`None` (ключ відсутній) і `[]` — різний сенс: `None` = перший запуск, `[]` = ініціалізовано але порожньо.

**Архітектура конфіг-driven:** нове джерело = один рядок у `DOCUMENT_SOURCES` у `documents.py`. Якщо структура HTML відрізняється — пишеться окремий парсер і вказується в полі `"parser"`.

**Реакції:** бот їх отримує але ігнорує (не тендерні повідомлення). В логах: "не знайдено в storage" — це нормально.

### Відомі обмеження

- **Пагінація mkrada.gov.ua** — перевіряється тільки перша сторінка (10 документів). Якщо між перевірками вийде більше 10 — надлишок пропуститься.
- **Хешування змін** — зміна вмісту існуючого документа не відстежується (навмисне рішення).
- **Окремі канали** — зараз всі джерела в один канал. Можна додати `chat_id` в конфіг кожного джерела.
